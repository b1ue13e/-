1. **先看动作**：从 [[🏠 主页]] 进入。
2. **再选方向**：学习内容先看 [[🧠 学习系统/🧭 学科分组导航|学科分组导航]]。

4. **最后执行**：按内容类型选择模板创建笔记或项目页。

## 🚀 怎么用

### 1. 学习内容
1. 打开 [[🧠 学习系统/🧭 学科分组导航|🧭 学科分组导航]] 判断自己当前处于哪一类。
2. 再进入 [[🧠 学习系统/🎓 学习入口|🎓 学习入口]] 查看具体路径、阶段页和专题支线。
3. 基于 [[📓 学习笔记模板]] 或 [[📓 快速笔记模板]] 创建笔记。
4. 补全 `concept`、`prerequisites`、`related`，让自动索引生效。

### 2. 项目内容
1. 基于 [[📁 项目模板]] 新建项目页。
2. 统一填写 `project_type`、`focus`、`domain`。
3. 在 [[📊 项目追踪]] 中查看焦点项目、已完成项目和分类统计。

### 3. 论文笔记
1. 打开 [[📚 论文库/📄 论文主页|📄 论文主页]]。
2. 将论文 PDF 放入 `📚 论文库/PDF库/`。
3. 基于 [[📄 论文笔记模板]] 创建论文笔记。
4. 用 `status: unread | reading | finished` 管理阅读状态。

### 4. 读书笔记
1. 打开 [[📚 书库/📚 书籍主页|📚 书籍主页]]。
2. 基于 [[📘 读书笔记模板]] 创建读书笔记。
3. 建议把书中的关键概念放入 `concept`，把方法迁移写入 `related`。

## 🧩 Frontmatter 约定

### 通用治理字段

| 字段 | 用途 |
|:--|:--|
| `type` | 页面类型，驱动 Dataview 查询 |
| `status` | 页面状态，按类型使用固定枚举 |
| `template: true` | 模板文件标记，避免统计时混入模板 |
| `domain` | 所属领域，便于后续做跨系统聚合 |

### 学习笔记
```yaml
---
type: study
template: true
topic: "Adam优化器"
created: 2026-04-20
status: ongoing
difficulty: intermediate
domain: ai
concept: ["梯度下降", "优化算法"]
prerequisites: ["梯度下降基础", "反向传播"]
related: ["SGD", "RMSprop"]
tags: [学习, 机器学习]
---
```

### 论文笔记
```yaml
---
type: paper
template: true
topic: "A Hybrid Attention Based ..."
created: 2026-04-20
year: 2024
status: reading
domain: paper
authors: "..."
venue: "Applied Energy"
concept: []
related: []
tags: [论文]
---
```

### 读书笔记
```yaml
---
type: book
template: true
title: "检索力"
created: 2026-04-20
status: reading
domain: book
category: "方法论"
topics: ["信息检索", "学习方法"]
concept: ["检索策略"]
related: ["论文检索"]
tags: [书籍, 阅读]
---
```

### 项目页
```yaml
---
type: project
template: true
topic: "项目名称"
created: 2026-04-20
status: active
project_type: learning  # learning | research | implementation | reading
focus: now  # now | next | later
domain: ai
deadline: 2026-05-01
progress: 0%
tags: [项目]
---
```

### 索引 / 说明页
```yaml
---
type: index
topic: "页面主题"
created: 2026-04-20
status: active
tags: [索引]
---
```

## ✅ 统一状态体系

- 学习页 / 阶段页：`planned | ongoing | completed`
- 项目页：`active | paused | completed`
- 论文页 / 读书页：`unread | reading | finished`
- 索引页：`active`

---

## 🔍 知识库体检

建议定期运行：

```powershell
.\scan_frontmatter.ps1
```

当前体检脚本会检查：

- 是否缺少 frontmatter
- `type` 是否在允许列表中
- `status` 是否匹配页面类型
- 关键字段是否缺失
- 模板页数量与治理字段覆盖率

---

## ✅ 维护建议

- 新建内容尽量都从模板开始。
- 项目页不要再混用学习页状态；统一用 `active / paused / completed`。
- 学习入口负责细节，主页只负责动作与总览。
- 路径越来越多时，优先使用 [[🧠 学习系统/🧭 学科分组导航|学科分组导航]] 降低认知负担。
- 发现统计异常时，先跑体检脚本，再修 frontmatter。

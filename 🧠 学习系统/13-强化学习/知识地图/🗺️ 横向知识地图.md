## 🎯 基础维度

> 强化学习的核心概念与数学框架

### 核心概念

| 概念 | 说明 | 关键公式/方法 |
|:---|:---|:---|
| **智能体-环境交互** | 智能体通过动作影响环境，环境返回状态与奖励 | $S_0, A_0, R_1, S_1, A_1, R_2, ...$ |
| **奖励 (Reward)** | 环境对智能体行为的反馈信号 | $R_t = r(S_t, A_t, S_{t+1})$ |
| **回报 (Return)** | 未来累积折扣奖励 | $G_t = \sum_{k=0}^{\infty} \gamma^k R_{t+k+1}$ |
| **探索 vs 利用** | Exploration-Exploitation 权衡 | ε-贪婪、UCB、汤普森采样 |

### MDP 建模

```
┌─────────────────────────────────────────────────────────┐
│                    MDP 五元组 ⟨S, A, P, R, γ⟩            │
├─────────────────────────────────────────────────────────┤
│  S: 状态空间   │   所有可能状态的集合                      │
│  A: 动作空间   │   所有可能动作的集合                      │
│  P: 转移概率   │   P(s'|s,a) - 状态转移动态                │
│  R: 奖励函数   │   R(s,a,s') 或 R(s,a)                    │
│  γ: 折扣因子   │   [0,1] 平衡即时与未来奖励                │
└─────────────────────────────────────────────────────────┘
```

### 关键区分

| 类型 | 特点 | 典型算法 |
|:---|:---|:---|
| **Model-based** | 学习/已知环境模型，进行规划 | 值迭代、策略迭代、MPC |
| **Model-free** | 直接从经验学习，无需环境模型 | Q-learning、SARSA、策略梯度 |
| **On-policy** | 只能使用当前策略采集的数据 | SARSA、REINFORCE、PPO |
| **Off-policy** | 可以使用历史数据或异策略数据 | Q-learning、DQN、DDPG |

## 💰 值函数维度

> 基于值函数的价值估计与优化

### 核心值函数

```
┌────────────────────────────────────────────────────────────────┐
│                      值函数定义                                 │
├────────────────────────────────────────────────────────────────┤
│                                                                │
│   状态值函数 V(s)        │        动作值函数 Q(s,a)              │
│   ─────────────────      │        ───────────────────           │
│   V^π(s) = E_π[G_t|S_t=s]│        Q^π(s,a) = E_π[G_t|S_t=s,A_t=a]│
│                        │                                       │
│   贝尔曼期望方程:         │        贝尔曼期望方程:                │
│   V^π(s) = Σ π(a|s) Σ p(s',r|s,a)[r + γV^π(s')]              │
│                        │        Q^π(s,a) = Σ p(s',r|s,a)[r + γΣ π(a'|s')Q^π(s',a')]│
│                        │                                       │
│   最优贝尔曼方程:         │        最优贝尔曼方程:                │
│   V*(s) = max_a Σ p(s',r|s,a)[r + γV*(s')]                   │
│                        │        Q*(s,a) = Σ p(s',r|s,a)[r + γmax_a' Q*(s',a')]│
│                        │                                       │
└────────────────────────────────────────────────────────────────┘
```

### 动态规划方法

| 算法 | 类型 | 核心更新 | 特点 |
|:---|:---|:---|:---|
| **策略评估 (Policy Evaluation)** | 预测 | 迭代计算 $V^π$ | 给定策略，估计值函数 |
| **策略迭代 (Policy Iteration)** | 控制 | 策略评估 + 策略改进交替 | 收敛快，每步需完全评估 |
| **值迭代 (Value Iteration)** | 控制 | $V_{k+1}(s) = \max_a \sum p(s',r \mid s,a)[r + \gamma V_k(s')]$ | 单步回溯，更高效 |
| **广义策略迭代 (GPI)** | 框架 | 评估与改进交替进行 | 大多数RL算法的底层框架 |

### 时序差分学习

```
TD 误差:  δ = R + γV(S') - V(S)

┌─────────────────┬─────────────────────────────────────┐
│    SARSA        │           Q-learning                │
│  (On-policy)    │          (Off-policy)               │
├─────────────────┼─────────────────────────────────────┤
│ Q(S,A) ← Q(S,A) │    Q(S,A) ← Q(S,A)                  │
│ + α[R + γQ(S',A')│   + α[R + γmax_a Q(S',a)            │
│ - Q(S,A)]       │         - Q(S,A)]                   │
├─────────────────┼─────────────────────────────────────┤
│ 实际采取的动作A' │     理论上最优的动作                 │
│ 保守、稳定      │     激进、高效                       │
└─────────────────┴─────────────────────────────────────┘
```

### 函数近似与深度RL

| 算法 | 值函数近似 | 特点 | 突破 |
|:---|:---|:---|:---|
| **DQN** | $Q(s,a;\theta)$ | 经验回放 + 目标网络 | 首个稳定深度RL |
| **Double DQN** | 双网络 | 解耦选择与评估 | 解决Q值过估计 |
| **Dueling DQN** | $V(s)+A(s,a)$ | 分离状态值与优势 | 某些状态所有动作价值相近时更高效 |
| **Prioritized Replay** | 优先级采样 | 按TD误差采样 | 提高样本效率 |

## 🎲 策略维度

> 直接参数化与优化策略

### 策略梯度基础

```
策略梯度定理:
∇J(θ) ∝ Σ_s μ(s) Σ_a ∇π(a|s;θ) Q^π(s,a)

          = E_π[∇log π(A|S;θ) · Q^π(S,A)]

┌─────────────────────────────────────────────────────────────┐
│  REINFORCE: 蒙特卡洛策略梯度                                 │
│  ∇J(θ) = E[G_t · ∇log π(A_t|S_t;θ)]                         │
│  ─────────────────────────────────────────────────────────  │
│  优点: 简单直接                                              │
│  缺点: 方差大，需要完整episode                                │
└─────────────────────────────────────────────────────────────┘
```

### Actor-Critic 架构

```
┌─────────────────────────────────────────────────────────────────┐
│                    Actor-Critic 框架                             │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│     ┌──────────┐         动作 a        ┌──────────┐            │
│     │          │ ───────────────────> │          │            │
│     │  Actor   │                      │  环境    │            │
│     │ (策略π)  │ <─────────────────── │          │            │
│     │          │    状态 s', 奖励 r   │          │            │
│     └────┬─────┘                      └──────────┘            │
│          │                                                      │
│          │ 更新策略 θ                                            │
│          │ ∇θ = α · δ · ∇log π(a|s;θ)                           │
│          │                                                      │
│          v                                                      │
│     ┌──────────┐         TD误差 δ         ┌──────────┐        │
│     │          │ <────────────────────── │          │        │
│     │  Critic  │                         │ 目标值   │        │
│     │ (值函数V)│  ────────────────────>  │ (Rt+γV') │        │
│     │          │                         │          │        │
│     └──────────┘                         └──────────┘        │
│          │                                                      │
│          └───────── 更新值函数 w ─────────────────────────>     │
│                      δ = R + γV(s') - V(s)                      │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 现代策略优化算法

| 算法 | 核心思想 | 关键技术 | 适用场景 |
|:---|:---|:---|:---|
| **A2C/A3C** | 异步/同步Actor-Critic | 并行环境采样 | 离散/连续动作 |
| **TRPO** | 信赖域策略优化 | KL散度约束保证单调改进 | 连续控制 |
| **PPO** | 近端策略优化 | Clipped surrogate objective | 连续/离散，最常用 |
| **SAC** | 软Actor-Critic | 最大熵正则 + 自动温度调节 | 样本高效、连续控制 |
| **IMPALA** | 分布式Actor-Critic | V-trace修正多步回报 | 大规模分布式训练 |

### PPO 核心公式

```
PPO-Clip 目标函数:
L^CLIP(θ) = Ê_t[min(r_t(θ)Â_t, clip(r_t(θ), 1-ε, 1+ε)Â_t)]

其中:
  r_t(θ) = π_θ(a_t|s_t) / π_θ_old(a_t|s_t)   (概率比)
  Â_t    = 优势函数估计 (可用GAE计算)
  ε      = 超参数 (通常 0.1 或 0.2)

直观理解:
  - 当优势为正，鼓励该动作，但概率比超过 1+ε 后停止更新
  - 当优势为负，抑制该动作，但概率比低于 1-ε 后停止更新
  - 防止策略更新过大，保证稳定性
```

### 多智能体强化学习 (MARL)

| 类型 | 架构 | 特点 | 代表算法 |
|:---|:---|:---|:---|
| **独立学习** | 每个智能体独立 | 将其他智能体视为环境一部分 | IQL、IA2C |
| **中心化训练去中心化执行 (CTDE)** | 训练时用全局信息，执行只用局部观测 | 平衡全局最优与可扩展性 | MADDPG、QMIX、MAPPO |
| **通信学习** | 学习何时/如何通信 | 智能体间协作/协调 | CommNet、TarMAC |
| **对手建模** | 建模其他智能体策略 | 博弈场景 | LOLA、NFSP |

### RLHF - 基于人类反馈的强化学习

```
RLHF 三阶段流程:

┌─────────────────┐    ┌─────────────────┐    ┌─────────────────┐
│   阶段1: SFT    │ -> │   阶段2: RM训练  │ -> │   阶段3: RL优化  │
│   监督微调      │    │   奖励模型训练   │    │   PPO优化策略    │
└─────────────────┘    └─────────────────┘    └─────────────────┘
        │                       │                       │
        ▼                       ▼                       ▼
   人工标注数据           人类偏好比较数据         策略生成 + RM评分
   (Prompt, Response)    (y_w > y_l | x)         + KL约束保持

应用: ChatGPT、Claude、InstructGPT 等对齐训练
```

### 模型预测控制 (MPC) + RL

| 方法 | 结合方式 | 优势 |
|:---|:---|:---|
| **MPC as Policy** | 用神经网络替代MPC中的优化求解器 | 实时性要求高的场景 |
| **MPC with Learned Model** | RL学习环境模型，MPC基于模型规划 | 利用MPC的安全性保证 |
| **MPC-Regularized RL** | RL策略 + MPC约束/终端代价 | 训练稳定性提升 |
| **MBPO** | 模型生成虚拟数据训练无模型RL | 样本效率大幅提升 |

### 其他前沿方向

| 方向 | 核心挑战 | 关键技术 |
|:---|:---|:---|
| **离线RL (Offline RL)** | 从固定数据集学习，无环境交互 | CQL、IQL、保守Q学习 |
| **层级RL (HRL)** | 长时程任务、稀疏奖励 | Option框架、FeUdal Networks |
| **元RL (Meta-RL)** | 快速适应新任务 | MAML、RL²、ProMP |
| **世界模型 (World Models)** | 学习环境动态进行规划 | Dreamer、PlaNet、MuZero |
| **安全RL** | 约束满足、安全探索 | CPO、Lagrangian方法 |
